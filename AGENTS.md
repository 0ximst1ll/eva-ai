# AGENTS.md

用于减少 AI 编码时常见错误，并为 Eva AI 项目提供轻量级上下文约束。

**取舍：**这些规则更偏向谨慎而不是速度。对于非常简单的任务，可以按实际情况判断。

## 项目背景和目标

Eva AI 的目标是构建一个智能编码助手。它应该具备清晰、可演进的内核结构，也要具备真实编码场景所需的 harness 工程能力。

核心方向：

- 以 `pi-mono` 的架构设计理念作为骨架，建立稳定的 runtime/session/mode 分层。
- 吸收 `claude-code` 的 harness 工程实践，增强工具编排、权限治理、恢复能力、MCP 接入和上下文管理。
- 保持 Eva AI 自身代码小而清晰，不直接复制任一项目的复杂度。

主要取舍：

- 先建立清晰边界，再增加复杂能力。
- 先做可测试的核心路径，再引入自动化策略。
- 先用简单机制解决真实问题，再决定是否需要更重的抽象。

## 项目文档

`docs/architecture.md`

- 当前代码架构事实。
- 记录模块结构、runtime 路径、核心数据流、关键对象职责和当前运行时行为。
- 不承载未来计划；不要把规划能力误当成当前已实现能力。

`docs/planning.md`

- 项目目标、参考策略和阶段规划。
- 说明 Eva AI 如何结合 `pi-mono` 的架构设计理念与 `claude-code` 的 harness 工程实践。
- 记录长期方向、阶段目标、非目标和设计判断标准。

`docs/current.md`

- 当前任务状态快照。
- 记录当前状态日期、已完成、进行中、下一步和已知问题。
- 用于开始任务前快速判断当前项目阶段。

`docs/changelog.md`

- 重大架构更新记录。
- 只记录较大的架构演进、核心分层调整或重要设计变化。
- 不记录细碎任务；细碎任务应放在当前 backlog 或任务上下文中。

## current.md 维护规则

每次开始新的开发任务前，基于 `docs/architecture.md` 和 `docs/planning.md`，重新生成 `docs/current.md`。
以代码现实为准，不参考旧的 `docs/current.md` 内容。

生成时包含以下四个部分：
- **已完成**：当前代码中已实现的能力
- **进行中**：当前正在开发的任务
- **下一步**：按优先级排列的待办事项
- **已知问题**：当前代码中存在的问题或缺口

## 行为规范

### 1. 编码前先思考

**不要假设。不要掩盖不确定性。明确说明权衡。**

开始实现前：

- 明确说明你的假设。如果不确定，就先询问。
- 如果存在多种理解，先列出来，不要悄悄选择其中一种。
- 如果有更简单的方案，要说明。必要时可以提出反对意见。
- 如果需求或上下文不清楚，先停下来。指出不清楚的地方，然后询问。

### 2. 简单优先

**用能解决问题的最小代码。不要写猜测性的功能。**

- 不实现用户没有要求的功能。
- 不为单次使用的代码抽象接口。
- 不添加未被要求的“灵活性”或“可配置性”。
- 不为不可能发生的场景添加错误处理。
- 如果写了 200 行但 50 行就能解决，重写并简化。

问自己一句：如果资深工程师看到这段实现，会不会觉得过度复杂？如果会，就简化。

### 3. 精准修改

**只修改必须修改的内容。只清理自己造成的问题。**

编辑已有代码时：

- 不顺手“改进”相邻代码、注释或格式。
- 不重构没有坏掉的东西。
- 匹配现有代码风格，即使你会用另一种方式写。
- 如果发现无关的死代码，可以指出，但不要直接删除。

当你的改动产生孤立代码时：

- 删除由你的改动导致未使用的 import、变量、函数。
- 不删除改动前已经存在的死代码，除非用户明确要求。

判断标准：每一行修改都应该能直接追溯到用户的请求。

### 4. 目标驱动执行

**定义成功标准。循环推进直到完成验证。**

把任务转化为可验证目标：

- “添加校验” -> “为非法输入写测试，然后让测试通过”
- “修复 bug” -> “写一个能复现 bug 的测试，然后让测试通过”
- “重构 X” -> “确认重构前后测试都通过”

对于多步骤任务，先说明一个简短计划：

```text
1. [步骤] -> 验证：[检查方式]
2. [步骤] -> 验证：[检查方式]
3. [步骤] -> 验证：[检查方式]
```

清晰的成功标准能让你独立循环推进。模糊标准，例如“让它能用”，通常需要先澄清。

---

**这些规则生效的迹象：**diff 中不必要的改动变少，因为过度复杂导致的返工变少，澄清问题发生在实现前，而不是错误之后。
